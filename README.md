# 界域物语 | NexusTale

如果你喜欢文字冒险，但又受不了固定选项、重复剧情和一眼望到头的体验，那界域物语就是为你准备的。

它想做的事情很简单：

把“说什么都可以”的自由，和“每一步都像真的在推进”的体验，放进同一款游戏里。

你可以和角色说话，可以临场做决定，可以改主意，可以绕路，可以冒险，也可以保守。

而游戏不会只是回你一段文字。

它会把你的选择变成路线、目标、事件、关系和画面，让这场旅程真正往前走。

---

## 它玩起来是什么感觉

最直接的感受，是“这趟冒险真的在发生”。

你不是点一下地点就瞬间抵达。

你会在地图上看到自己正在前往哪里，离目标还有多远，途中发生了什么。

你不是进了一个场景，说几句话就结束。

你会进入建筑、推进探索、发现新地点、接到目标、看到进度变化，知道自己不是停在原地聊天，而是在一段旅程里不断深入。

你也不会总是得到最顺利、最讨喜、最安全的结果。

有时候你会判断失误。

有时候你会被迫转向。

有时候你刚觉得局面稳了，新的麻烦就已经在路上。

这种不确定感，才是它像游戏的地方。

---

## 为什么很多人会愿意继续玩下去

### 1. 因为这个世界不是摆设

界域物语不是把“地图”当作一张配图挂在旁边。

地图、地点、建筑、任务目标和移动进度，都会持续参与游戏过程。

你能清楚知道自己在哪里、要去哪、已经推进到哪一步。

这会让探索变得更具体，也让每次抵达新地方都更有意义。

### 2. 因为它不只是让你读故事，而是让你做选择

很多互动体验的问题在于，看起来自由，实际没有重量。

你说了很多，局面却没怎么变。

界域物语更在意另一件事：

当你决定继续深入、立刻撤离、先去办事、还是转头探索别的区域时，游戏会真的给出不同走向。

你不需要先理解它背后怎么运作。

你只会直接感觉到一件事：

**你的选择是有后果的。**

### 3. 因为它会持续给你“正在推进”的反馈

目标提示、地图总览、探索进度、旅途进度、新发现、新任务、阶段性通知，这些东西放在一起，会让你很容易进入状态。

你不用一边玩一边自己脑补“现在到底推进到哪了”。

界域物语会把这件事清楚地展示出来。

这对沉浸感非常重要。

因为玩家真正想要的，不是复杂，而是明确。

### 4. 因为它不只靠文字撑场面

角色立绘、世界地图、实时场景图、动态氛围音乐，这些内容不是为了堆功能而存在。

它们的作用，是帮你更快进入这个世界，也帮你更容易记住这次旅程。

当一段经历不再只停留在一屏字里，它就更容易变成你真的玩过的一次冒险。

### 5. 因为它会让你在意同行的人

很多人以为文字冒险最重要的是事件，其实不是。

真正让人舍不得退出的，往往是关系。

是你开始记住某个角色怎么说话，开始在意某个人会不会受伤，开始想知道这段同行最后会走到哪里。

界域物语想留下来的，就是这种感觉。

---

## 它最适合哪种玩家

- 喜欢故事驱动，但不想被固定选项牵着走的人
- 喜欢探索地图、推进目标、解锁新地点的人
- 喜欢角色互动，希望旅途里有人可记、可念、可牵挂的人
- 喜欢沉浸感，希望画面、音乐、文字是一起服务体验的人
- 喜欢每次开局都有新鲜感，不想反复看同一套剧情模板的人

---

## 这款游戏现在已经成型到什么程度

如果把它当成一个完整产品来看，界域物语现在已经具备了一套比较完整的体验闭环：

从开场创建角色，到生成世界观和视觉风格；

从进入地图，到选择路线、抵达地点、推进探索；

从接到任务，到发现新区域、触发关键节点、积累关系；

从文字推进，到图片生成、通知演出、音乐切换和状态反馈。

它不再像一个只有核心想法的原型。

它更像一款已经开始认真思考“玩家为什么愿意继续留下来”的游戏。

这也是为什么 README 不应该再写成技术笔记。

因为它现在更值得被当成一款作品来介绍。

## 你会在里面遇到什么

你会遇到要不要继续往前走的犹豫。

会遇到刚刚抵达新地点时的期待。

会遇到终于把一片区域摸透之后的轻松。

会遇到计划突然被打断时的狼狈。

会遇到某个角色一句话就让整个旅途气氛变掉的时刻。

也会遇到那种“再往前走一点看看”的冲动。

如果一款游戏能不断制造这种冲动，它就已经有足够强的吸引力。

界域物语想做的，就是把这种吸引力，尽可能稳定地交到玩家手里。

## 现在启程

启动方式放在最后，因为它确实不是这份 README 最重要的部分。

如果你只是想尽快进入界域物语，步骤很短：

1. 克隆项目

```bash
git clone https://github.com/hewenhan/ai-text-adventure-engine.git
cd ai-text-adventure-engine
```

2. 安装依赖

```bash
npm install
```

3. 配置 `.env`

```env
VITE_GEMINI_API_KEY=你的密钥
```

4. 启动游戏

```bash
npm run dev
```

浏览器打开 `http://localhost:3000`，然后出发。
